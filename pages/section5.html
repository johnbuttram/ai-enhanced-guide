Section 5: Navigation System (Desktop & Mobile)

One key feature is a unified navigation menu that works on both desktop and mobile:

Desktop Sidebar

On medium and larger screens (md: breakpoint and up), show a sidebar on the left side of the screen containing the site title and navigation links.

Use an <aside> element for the sidebar, with an appropriate label (e.g., aria-label="Main Sidebar Navigation").

<aside class="hidden md:flex md:flex-col md:w-64 md:h-screen bg-gray-800 p-4 text-white print:hidden" aria-label="Main Sidebar Navigation">

The sidebar is hidden by default and becomes visible on medium+ screens. Use high contrast colors and logical heading structure. Consider <header> or role="heading" wrappers for semantic structure.

Clicking the overlay or the menu button should close the drawer. The drawer mirrors the desktop nav. Maintain accessibility roles like role="navigation" and ARIA labels.

Navigation Behavior

  • Sidebar is fixed or scrolls with page (decide consistently).
  • Apply md:ml-64 margin to <main> if using fixed sidebar.
  • Use aria-current="page" for current page indicator.
  • Ensure skip link targets #main and is focusable.
  • Use consistent hover:text styles and padding across links.
  • Ensure mobile nav resets on page load (menu closed by default).
